# Prepare Your Room Station

Use this page before guests enter the room. It covers the operational checks that matter at the station where you run the Room Dashboard.

## Pre-Session Checklist

Before starting a real session, confirm:

* The correct room is open from `All Rooms`
* Live View opens on the correct player-facing display
* The dashboard can be used comfortably at your station
* Key room media is available
* The cache status is not showing an unexpected warning
* The room has been reset to a clean start state

## Open Live View And Verify The Display

1. Open the room from `All Rooms`.
2. Click `Open Live View`.
3. Confirm the player-facing display shows the expected room.

Check:

* The background is correct
* The timer is readable from the room
* The live message area appears where expected
* The clue counter is visible only if intended
* Success and failure screens were configured in advance

If the preview looked different in the editor, use the real display as the final source of truth and adjust the room setup before going live.

## Use Fullscreen When It Helps

Use `Fullscreen` on the Room Dashboard when you want:

* Fewer browser distractions
* A larger control surface for game masters

If you are sharing the computer for multiple tasks, you may prefer to leave fullscreen off until the room starts.

## Check Media Before Players Arrive

Use the `Audio Mixer` and available media controls to verify:

* Alert tone is present
* Soundtrack is present if the room uses one
* Uploaded audio can be previewed
* Room-specific media is available for use during the session

If the room relies on video, image, or audio cues, verify those assets before the session starts instead of assuming they are ready.

## Understand Cache Status

The cache status indicator tells you whether room media is ready for offline use on the current station.

### Offline Ready

`Offline Ready` means the required media has been cached successfully for that room. This is the best state to be in before live play.

### Caching or Preparing Cache

This means the station is still preparing room media. If possible, leave the room open until caching completes.

### Cache Warning

`Cache Warning` means some room media failed to cache or caching is unavailable in the current environment. This does not always mean the room is unusable, but you should verify all required media manually before starting.

## Final Ready Check

Before players enter:

* Reset the room if needed
* Confirm the starting timer and clue count are correct
* Confirm Live View is open on the correct screen
* Confirm media assets needed early in the session are available
* Confirm the game master can clearly see and use the Room Dashboard

## If Something Looks Wrong

If the station is not ready, stop and fix setup before starting a live run. Use [Troubleshoot Live Session Issues](troubleshoot-live-session-issues.md) for common problems.
